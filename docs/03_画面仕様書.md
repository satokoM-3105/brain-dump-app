## 3. メイン画面（`index.html`）

### 3.1 画面名

**メイン画面**（認証とメモ機能を同一 HTML で提供）

### 3.2 画面の目的

- 未ログイン時: メールアドレスとパスワードで **新規登録** または **ログイン** する
- ログイン・登録成功後: **同一 `index.html` 内** でメモ機能エリアに切り替える
- ログイン後: メール表示・ログアウト、メモの CRUD を行う
- 認証エラーは **`alert()`** で表示する

### 3.3 画面レイアウト（未ログイン時）

```
┌─────────────────────────────────────┐
│  アプリタイトル                        │
├─────────────────────────────────────┤
│  メールアドレス入力欄                   │
│  パスワード入力欄                      │
│  [ ログイン ]  [ 新規登録 ]            │
└─────────────────────────────────────┘
```

### 3.4 画面レイアウト（ログイン後）

```
┌─────────────────────────────────────┐
│  アプリタイトル                        │
│  ログイン中: user@example.com [ログアウト] │
├─────────────────────────────────────┤
│  メッセージ表示エリア                    │
├─────────────────────────────────────┤
│  タイトル入力欄                        │
│  カテゴリ選択（ドロップダウン）           │
│  本文入力欄                            │
│  [ 保存ボタン ]                        │
├─────────────────────────────────────┤
│  メモ一覧エリア                        │
│    ・メモ1（タイトル・カテゴリ・日時…）   │
│      [ 編集 ] [ 削除 ]                 │
│    ・メモ2 …                          │
└─────────────────────────────────────┘
```

- PC では横幅をある程度に抑え、中央寄せにして読みやすくする
- スマホでは画面幅いっぱいを使い、縦にスクロールできるようにする

最低限、次の要素を画面に置く。

| No. | 要素名 | 種類（HTMLの目安） |
|-----|--------|-------------------|
| 1 | アプリタイトル | 見出し（`h1` など） |

| 3 | パスワード入力欄 | `input type="password"`（未ログイン時） |
| 4 | ログインボタン | ボタン（未ログイン時） |
| 5 | 新規登録ボタン | ボタン（未ログイン時） |
| 6 | ログインメール表示 | テキスト（ログイン後） |
| 7 | ログアウトボタン | ボタン（ログイン後） |
| 8 | メッセージ表示エリア | テキスト表示用の領域（`div` など） |
| 9 | タイトル入力欄 | 1行テキスト入力（`input type="text"` など） |
| 10 | カテゴリ選択（ドロップダウン） | プルダウン（`select`） |
| 11 | 本文入力欄 | 複数行テキスト入力（`textarea` など） |
| 12 | 保存ボタン | ボタン（`button`） |
| 13 | メモ一覧エリア | 一覧を入れるコンテナ（`div` など） |

| 15 | 削除ボタン | 各メモ行に付けるボタン |

### 3.6 各要素の役割

| 要素名 | 役割 |
|--------|------|
| アプリタイトル | アプリ名「ブレインダンプ」を表示し、何のアプリか一目で分かるようにする |
| メールアドレス入力欄 | ログイン・新規登録用。未ログイン時のみ表示 |
| パスワード入力欄 | ログイン・新規登録用（6 文字以上）。未ログイン時のみ表示 |
| ログインボタン | `signInWithPassword`。成功後メモ機能エリアへ切替 |
| 新規登録ボタン | `signUp`。成功後メモ機能エリアへ切替 |
| ログインメール表示 | Supabase Auth セッションのメールアドレスを表示する |
| ログアウトボタン | 押下で `signOut` し、認証エリアを表示する |
| メッセージ表示エリア | 保存成功・更新成功・削除成功・入力エラー・接続エラーなどを短い文で表示する。通常時は空または非表示でもよい |
| タイトル入力欄 | メモのタイトルを入力する。新規作成時も編集時も同じ欄を使う |
| カテゴリ選択（ドロップダウン） | マスタテーブル `categories` から取得した選択肢を表示する。未選択用の先頭項目（例：「カテゴリを選ぶ」「未分類」）を置いてもよい。**必須ではない** |
| 本文入力欄 | メモの本文を入力する。新規作成時も編集時も同じ欄を使う |
| 保存ボタン | 新規のときは「保存」、編集中のときは「更新」として同じボタンで DB に反映する（ラベルは実装で「保存」のままでも、「保存／更新」に切り替えてもよい） |
| メモ一覧エリア | Supabase から取得したメモをリスト形式で表示する |
| 編集ボタン | 押したメモのタイトル・本文・カテゴリを入力欄に読み込み、編集モードにする |
| 削除ボタン | 押したメモを削除する（確認ダイアログのあと実行） |

## 5. ボタン一覧

| ボタン名 | 表示場所 | 押したときの動き（概要） |
|----------|----------|--------------------------|
| ログインボタン | 認証エリア | `signInWithPassword`。成功 → メモ機能表示 |
| 新規登録ボタン | 認証エリア | `signUp`。成功 → メモ機能表示 |
| ログアウトボタン | メモ機能上部 | `signOut` 後、認証エリアを表示 |
| 保存ボタン | 入力欄の下 | 新規なら INSERT、編集中なら UPDATE。成功後は入力欄を空にし、一覧を再表示 |
| 編集ボタン | 各メモ行 | そのメモのタイトル・本文・カテゴリを入力欄に表示し、編集中のメモ ID を内部で保持 |
| 削除ボタン | 各メモ行 | 確認ダイアログを出し、OK なら DELETE。成功後は一覧を再表示 |

※ ブラウザ標準の「確認ダイアログ」（`confirm`）を使う想定。独自のモーダル画面は今回は必須としない。

## 6. 入力項目一覧

| 項目名 | 必須 | 入力形式 | 最大目安 | 備考 |
|--------|------|----------|----------|------|
| メールアドレス | はい（認証時） | email | — | 空白不可 |
| パスワード | はい（認証時） | password | — | **6 文字以上** |
| タイトル | はい | 1行テキスト | 実装で妥当な長さ（例：数百文字） | 空白のみは不可 |
| カテゴリ | **いいえ** | ドロップダウン（`select`） | — | マスタ `categories` の一覧から選択。未選択可 |
| 本文 | はい | 複数行テキスト | 実装で妥当な長さ | 空白のみは不可 |

**入力チェック（画面上）**

- 保存ボタン押下時、タイトルまたは本文が空（前後の空白だけも空とみなす）の場合は保存せず、メッセージ表示エリアに入力エラーを出す
- カテゴリは未選択でも保存・更新してよい（必須チェックは行わない）

## 8. 画面の状態

1画面ですが、**画面上の見え方と操作の違い**として次の状態を定義する。

| 状態名 | いつその状態になるか | 画面上の見え方・操作 |
|--------|----------------------|----------------------|
| 未ログイン | セッションなし | 認証エリア（メール・パスワード・ログイン・新規登録）のみ表示 |
| ログイン済み | セッションあり | メモ機能エリアを表示。メール表示・ログアウト可能 |
| 初期表示（ログイン後） | ログイン直後 | 入力欄は空。一覧を読み込み中、または読み込み完了後にメモ一覧を表示 |
| 入力中 | タイトル・本文のいずれかに文字がある | 保存ボタンが押せる。メッセージは通常クリアまたは前回メッセージのまま |
| 保存成功 | 新規保存が成功した直後 | 入力欄が空になる。一覧が更新される。成功メッセージを表示 |
| 保存失敗 | 新規保存が失敗したとき | 入力欄の内容は残す。エラーメッセージを表示 |
| 一覧表示 | データ取得後、特に編集していないとき | メモが新しい順に並ぶ。各行に編集・削除ボタン |
| 編集中 | 編集ボタンを押したあと | 入力欄に選択メモのタイトル・本文・カテゴリが入る。内部で編集中のメモ ID を保持。保存ボタンは更新として動作 |
| 更新成功 | 編集内容の更新が成功した直後 | 編集中 ID をクリア。入力欄を空にする。一覧を再表示。成功メッセージ |
| 削除確認 | 削除ボタンを押した直後 | ブラウザの確認ダイアログ「削除しますか？」など（OK / キャンセル） |
| 削除成功 | 削除が成功した直後 | 一覧から該当メモが消える。成功メッセージ |
| エラー発生時 | 取得・保存・更新・削除のいずれかが失敗 | エラーメッセージを表示。可能なら入力内容は残す |

**状態の遷移（簡易）**

```
初期表示 → 一覧表示
入力中 → 保存成功 → 一覧表示
入力中 → 保存失敗 → 入力中（内容保持）
一覧表示 → 編集中 → 更新成功 → 一覧表示
一覧表示 → 削除確認 → 削除成功 → 一覧表示
任意 → エラー発生時
```

## 9. スマホ表示で注意すること

| 項目 | 方針 |
|------|------|
| 画面幅 | 320px 前後の狭い幅でも横スクロールが出ないようにする |
| タップ領域 | 保存・編集・削除ボタンは指で押しやすい大きさ（高さ・余白を十分に） |
| 入力欄 | タイトル・本文・カテゴリは画面幅いっぱいに近い幅で表示 |
| カテゴリ | `select` もタップしやすい高さ（16px 前後のフォント）にする |
| 本文入力 | `textarea` の行数はスマホでも十分に確保（例：5行以上） |
| 一覧 | 1件が縦に長くなりすぎないよう、本文プレビューは省略表示を検討 |
| フォント | 本文は小さすぎないサイズ（例：16px 前後）で読みやすく |
| 確認ダイアログ | スマホでもブラウザ標準の `confirm` で問題ない |

## 10. 使用するデータ（画面とDBの対応）

テーブル定義の詳細は **`docs/02_DB仕様書.md`** を参照。

### 10.1 `brain_dumps`

| カラム名 | 画面での扱い |
|----------|--------------|
| `user_id` | INSERT 時に DB トリガで `auth.uid()` をセット（画面では非表示） |
| `id` | 一覧・編集・削除時に内部で保持（ユーザーには見せなくてよい） |
| `title` | 入力欄・一覧表示 |
| `content` | 入力欄・一覧プレビュー |
| `category_id` | ドロップダウンの選択値。未選択時は NULL |
| `created_at` | 一覧に日時として表示 |

### 10.2 `categories`（マスタ）

| カラム名 | 画面での扱い |
|----------|--------------|
| `id` | ドロップダウンの `option` の value |
| `name` | ドロップダウン・一覧の表示ラベル |

※ `categories` はログイン前後どちらからも SELECT 可能（RLS ポリシー参照）。

## 13. カテゴリ UI 仕様（FR-07 対応）

要求仕様 **FR-07** に基づく画面実装の要点。

| 項目 | 仕様 |
|------|------|
| 配置 | タイトル入力欄の直下、本文入力欄の上 |
| コントロール | `<select id="category">` |
| 先頭 option | `value=""`、表示文言「カテゴリを選ぶ（任意）」など |
| 選択肢の取得 | ページ読み込み時に `categories` テーブルを SELECT（固定リスト禁止） |
| 必須 | **いいえ**。先頭 option のまま保存可 → DB では `category_id = NULL` |
| 編集時 | 既存メモの `category_id` に一致する option を選択。NULL のときは先頭 option |
| 一覧表示 | カテゴリ名をタイトル付近に表示。未設定は **「未分類」** |
