# ブレインダンプアプリ — 要求仕様書

## 1. 概要

### 1.1 目的

思考・アイデアを手軽に記録し、後から一覧・編集・削除できる**自分専用のメモアプリ**を提供する。バックエンドは **Supabase** を用い、認証により他ユーザからデータを分離する。

### 1.2 想定利用者

アプリの利用者は**本人のみ**（シングルユーザー用途を想定したマルチテナント設計：ログインユーザごとにデータが分離される）。

### 1.3 利用環境

- **スマートフォン**および**PC**のブラウザから利用できること（レスポンシブなWeb UI）。
- オフライン必須要件は本書の対象外とする（オンライン前提）。

---

## 2. 用語

| 用語 | 説明 |
|------|------|
| メモ | ユーザーが入力した思考・アイデアの1件。DB上は `brain_dumps` の1レコードに対応。 |
| ブレインダンプ | 頭に浮かんだ内容を加工せずに書き出す行為。本アプリではテキストとして保存する。 |
| カテゴリ | メモを分類するためのラベル。選択肢はマスタテーブル（`categories`）で管理する。 |

---

## 3. 機能要求

### 3.1 メモの作成・保存（FR-01）

- ユーザーは**タイトル**と**本文**をテキストで入力できる。
- タイトル・本文はいずれも**必須**とする（空のまま保存不可）。
- 保存操作により、サーバ上に新規レコードが作成される。
- 作成日時はサーバ側で自動設定される（クライアントの改ざんに依存しない）。

### 3.2 メモ一覧（FR-02）

- ログイン中のユーザーは、**自分が保存したメモ**を一覧で閲覧できる。
- 一覧には少なくともタイトルおよび作成日時が分かる表示とする（本文のプレビューは実装の自由とする）。
- 並び順のデフォルトは**作成日時の新しい順**とする（別順を将来追加してもよい）。

### 3.3 メモの編集（FR-03）

- ユーザーは一覧または詳細から既存メモを選び、タイトル・本文を**編集**できる。
- 編集後、保存するとサーバ上の該当レコードが更新される。
- **他ユーザのメモは編集できない**こと。

### 3.4 メモの削除（FR-04）

- ユーザーはメモを**削除**できる。削除後は一覧・検索対象から除外される。
- **他ユーザのメモは削除できない**こと。

### 3.5 認証・データの分離（FR-05）

- **ログイン**したユーザーのみ、アプリの主要機能（作成・一覧・編集・削除）を利用できる。
- 未ログイン時はログインを促す画面またはリダイレクトとする（公開共有の閲覧要件はない）。
- データベース上では、**認証ユーザ（Supabase Auth）とメモの紐づけ**により、誰のメモかを識別し、**自分のデータのみ**が参照・更新・削除の対象となること（Row Level Security 等で担保）。

### 3.6 マルチデバイス（FR-06）

- 同一アカウントでスマホ・PCのいずれからログインしても、**同じメモ集合**が見えること（クラウド同期）。

### 3.7 カテゴリ管理（FR-07）

- メモに**カテゴリ**を設定できる。
- カテゴリの選択肢は、**専用のマスタテーブル**（`categories`）で管理する。
- メモの**新規投稿・編集**のとき、画面上の**ドロップダウン**（プルダウン）からカテゴリを選べる。
- カテゴリの入力は**必須としない**（未選択のまま保存できる）。
- メモ**一覧**には、設定されている場合は**カテゴリ名**を表示する（未設定の場合は「未分類」など分かる表示とする）。

---

## 4. 非機能要求

### 4.1 セキュリティ

- 通信はHTTPSを前提とする（Supabase / ホスティング環境の標準構成に準拠）。
- API・DB は認証トークンに基づきアクセス制御されること。

### 4.2 可用性・性能

- 個人利用を想定し、詳細なSLOは定めない。通常のオンライン操作が実用的な応答時間であること。

### 4.3 データモデルに関する方針

- メモ本体は `brain_dumps`、カテゴリの選択肢は `categories` マスタで管理する。
- 認証ユーザとの紐づけ用カラム等の詳細は **DB仕様書** に従う。

---

## 5. 対象外

- メモの共同編集・共有URL・公開ノート機能。
- リッチテキスト専用エディタ、添付ファイル、タグ・フォルダ階層、**カテゴリマスタの画面からの追加・編集・削除**（将来拡張の余地は残せる）。
- ネイティブアプリ（モバイルストア配布）は本書では必須としない（PWA化は実装次第）。

---

## 6. 受入基準（要約）

| ID | 内容 |
|----|------|
| AC-01 | ログイン後、タイトル・本文を入力してメモを保存でき、一覧に表示される。 |
| AC-02 | 一覧からメモを開き編集でき、変更が反映される。 |
| AC-03 | メモを削除でき、一覧から消える。 |
| AC-04 | 別ユーザーでログインした場合、他ユーザのメモは見えない・操作できない。 |
| AC-05 | スマホ幅・PC幅の画面で致命的なレイアウト崩れなく操作できる。 |
| AC-06 | メモ投稿・編集時にドロップダウンでカテゴリを選べる（未選択でも保存できる）。 |
| AC-07 | 一覧にカテゴリ名（または未分類）が表示される。 |
